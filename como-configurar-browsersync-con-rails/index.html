Como configurar BrowserSync con Rails

BrowserSync es un excelente herramienta a la hora de realizar desarrollo frontend, ya que nos permite poder realizar pruebas en distintos dispositivos, y refresca el navegador luego de editar y salvar los archivos que se le indiquen, por ejemplo los archivos html, css, js, etc...

En este post te voy a enseñar como utilizar BrowserSync con Rails, lo cual es muy sencillo.

Lo primero que necesitamos tener instalado es Nodejs y BrowserSync, para ello instalaremos Nodejs, descargando el instalador en la página oficial, tambien podemos instalarlo vía Homebrew si nos encontramos en un computador Mac brew install node.

Luego necesitaremos instalar BrowserSync, lo haremos a travez de NPM, el gestor de paquetes de Nodejs npm install -g browser-sync, lo instalaremos de forma global -g para poder utilizarlo en cualquier proyecto que necesitemos.

Lo siguiente será iniciar el servidor de rails rails s.

Y indicarle a BrowserSync, que utilice como proxy, la URL que nos sirve para ver nuestro proyecto de rails en el navegador que por lo general es localhost:3000.

Iniciamos BrowserSync

browser-sync start --proxy localhost:3000 --files "app/assets/stylesheets/**/*.scss, app/views/**/*.html.erb"

¡Y eso es todo! Veras la siguiente salida de seguimiento en la terminal:

[BS] Proxying: http://localhost:3000
[BS] Access URLs:
 ----------------------------------
       Local: http://localhost:3001
    External: http://10.0.1.3:3001
 ----------------------------------
          UI: http://localhost:3002
 UI External: http://10.0.1.3:3002
 ----------------------------------
[BS] Watching files...

Ahora podremos revisar nuestro trabajo en diferentes dispositivos por medio de la URL externa http://10.0.1.3:3002, y nuestro navegador se refrescara automáticamente si realizamos cambios en nuestros archivos SCSS y HTML.
